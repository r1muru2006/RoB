Detection Thresholds

Per-file-type thresholds & reference data

Paper §6.1 measured Δ entropy and Δ size on 1,000,000 modifications (500K benign + 500K malicious) across 5,000 files of 5 file types. Encryption barely changes file size (AES only adds a small fixed overhead) but dramatically raises entropy for low-entropy formats. Already-compressed formats (xlsx, pdf, jpeg) start near max entropy, so the encryption signal is much weaker there.

Detection rule: a write is flagged when entropy Δ exceeds the threshold AND size Δ stays below it. Both conditions must hold — this is what makes encryption distinguishable from benign edits, which usually shift size by ~15% with negligible entropy gain.
