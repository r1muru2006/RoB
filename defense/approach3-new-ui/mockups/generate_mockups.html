New UI Design for FSA API Permission Dialogs

Based on Section 6.3 of "RoB: Ransomware over Modern Web Browsers" (USENIX Security 2023)

Figure 4: Read Access Permission Box (Old and New)

Old UI
Let site view files?
www.example.com will be able to view files in directory_name until you close all tabs for this site.
New UI
Let site view files?
⚠
Warning! www.example.com will be able to read all files in directory_name and its subdirectories until you close all tabs for this site.
www.example.com might attempt to steal your sensitive information.

Design Decisions (from Section 6.3)

  • The interface must explicitly show the risks (sensitive information disclosure, permanent data loss)
  • Dialog boxes must be designed with warning colors and icons to capture attention
  • Permission dialog boxes must not be identical so users know the difference
  • The new UI must show all accessed/changed files for meaningful user decisions
  • A link to report suspicious websites helps build a community blocklist
